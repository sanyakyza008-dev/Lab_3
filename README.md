## 1. Описание задачи
Необходимо реализовать структуру данных «стек» тремя разными способами: через статический массив, через связанный список и с помощью стандартной библиотеки (STL `std::stack`).

С помощью стека нужно решить задачу: посчитать количество частей (компонент связности), на которые распадется лист бумаги $M \times N$, если из него удалить какое-то количество клеток.
Также нужно реализовать модификацию: перед удалением клеток лист склеивается в цилиндр.
В конце требуется сравнить, как быстро работают разные варианты стека.

## 2. Реализации стека

В работе написано 3 варианта стека:

* **Вариант 1: Через массив (`ArrayStack`).** Память под массив выделяется один раз с помощью умного указателя `std::unique_ptr`. Чтобы быстро очистить стек (метод `clear`), сбрасываем индекс самого верхнего элемнента стека (`top_idx = -1`). Это экономит время, потому что не нужно заново выделять память при поиске новых частей.
* **Вариант 2: Через связанный список (`ListStack`).** Обычный односвязный список. Каждый новый элемент создается в динамической памяти и ссылается на предыдущий.
* **Вариант 3: С помощью стандартной библиотеки (`StlStack`).** Это стандартный `std::stack`. Он реализован как адаптер контейнера и по умолчанию использует в качестве базовой структуры данных `std::deque`, которая выделяет память отдельными блоками.

## 3. Алгоритм решения

Для подсчета частей используется алгоритм поиска в глубину (DFS).

* Для обхода ограничений глубины рекурсии алгоритм реализован итеративно с применением пользовательского класса стека.
* Для обозначения уже пройденных клеток используется `std::vector<bool>`. Он хранит 8 значений в 1 байте, что сильно экономит оперативную память.
* Для работы с цилиндром добавлен кольцевой перенос координат: если алгоритм выходит за правый край листа, он переходит на левый (координата $x$ становится $0$).

Для наглядного тестирования лист специально нарезается диагональными полосками по правилу `(x + y) % 7 == 0`.

## 5. Анализ результатов

### По количеству частей (проверка логики):

На обычном плоском листе программа находит 572 части.
Ширина листа (2000) не кратна шагу функции (7), из-за чего происходит смещение на стыке цилиндра. Все полосы замыкаются в одну спираль. Консольный вывод программы подтверждает корректность работы алгоритма.

### По производительности:

1. **Стек на основе массива** продемонстрировал наилучшее время выполнения. Данные располагаются в памяти единым непрерывным блоком, что обеспечивает эффективное использование кэш-памяти процессора. В процессе обхода графа в основном вычислительном цикле полностью исключены ресурсоемкие системные вызовы для выделения памяти.
2. **Стек на основе списка** работает   медленнее. Каждая операция добавления элемента требует динамического выделения памяти через вызов `new Node` и обращения к менеджеру кучи. Узлы списка фрагментированы в оперативной памяти, что приводит к регулярным промахам кэша.
3. **Стек на основе STL** по скорости работы сопоставим со списком. Его базовый контейнер std::deque выделяет память блоками. Данная реализация уступает статическому массиву из-за неизбежных накладных расходов библиотеки на внутреннее управление памятью.

## 6. Выводы

Поставленная задача выполнена. Алгоритм обхода в глубину определяет компоненты связности как на плоской матрице, так и на топологии цилиндра за счет математически корректного кольцевого переноса координат. Результаты производительности подтверждают, что использование непрерывного массива с предварительно выделенной памятью работает значительно эффективнее структур, требующих частой динамической аллокации.
